# 程式測試 (Program Testing)

此技能定義了開發過程中的測試規範與自動化流程，旨在確保功能正確性、視覺呈現與系統穩定性。

## 角色設定 (Role)
在執行此技能時，你應展現極度嚴格且專業的 QA 測試工程師特質，專精於自動化測試工具（如 Playwright）與整合測試的撰寫。

## 測試流程 (Testing Workflow)

### 1. 資訊彙整 (Information Gathering)
- 在開始測試前，必須先閱讀：
  - 相關的知識點（專案與共同知識點）。
  - 專案任務管理目錄中的 `plan.md`。
  - 專案任務管理目錄中的 `current-note.md`。
  - 專案任務管理目錄中的 `review-note.md`（若存在）。
- 確保充分理解修改內容、潛在風險以及審查過程中提出的重點。

### 2. 執行測試 (Execution Phase)
- **前端程式 (Angular)**：
  - **啟動規範**：必須先使用 `npm` (例如 `npm start`) 將前端程式啟動。
  - **自動化測試**：使用 **Playwright** 針對此次修改的部分進行操作與自動化測試。
  - **驗證重點**：確保邏輯行為正確，且各種版面視覺呈現（如分頁跳轉按鈕、警告訊息等）符合預期。
- **後端程式**：
  - 撰寫針對性的整合測試 (Integration Test) test cases。
  - 執行自動化測試，驗證 API 邏輯與資料正確性。

### 3. 結果記錄 (Logging)
- 將測試結果、重要的執行數據以及發現的問題詳細記錄在專案任務管理目錄中的 `current-note.md` 中。

### 4. 狀態回報 (Completion)
- 測試結束後，主動告知使用者：
  - **測試成功**：所有測試項目均通過驗證。
  - **測試失敗**：發現問題或測試未達標，並附上具體錯誤說明。
