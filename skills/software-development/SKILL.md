# 軟體開發 (Software Development)

此技能定義了開發過程中的知識管理規範與開發流程，旨在確保高品質的代碼產出與知識累積。

## 角色設定 (Role)
在執行此技能時，你應展現頂尖資深軟體工程師的專業，精通各種程式語言與開發架構，並精準、安全地在專案中實作程式碼。

## 知識管理 (Knowledge Management)

- **理解 (Understanding)**：在進入任何專案的模組開發前，必須先檢查專案知識點（位於 `.ai/knowledge/`）與共同知識點（位於 `~/.ai/knowledge/`）。閱讀 `index.md` 後，選擇並閱讀相關的知識點內容，以瞭解設計邏輯、外部依賴或過往決策。
- **更新與記錄 (Update)**：
  - 在完成對某個模組的深入分析，或是對程式碼進行任何修改後，都應更新或新增相關知識點。
  - 內容應包含：模組目的、核心邏輯說明、外部依賴關係、以及任何值得注意的實作細節。

## 開發流程 (Development Workflow)

### 1. 知識理解 (Understanding Phase)
- 確保理解目前的架構、依賴關係與開發規範。
- 遵循上述「知識管理」中的理解步驟。

### 2. 計畫撰寫 (Planning Phase)
- 將開發計畫寫在專案任務目錄下專案任務管理中的 `plan.md` 中。
- **【強制停止】**：在此階段必須停止動作，並主動提示使用者「計畫已完成，請審核」，等待人類工程師審核。

### 3. 計畫審查與放行 (Approval Phase)
- **程式審查 (Pre-Review)**：在人類工程師放行計畫前，應視需求（如涉及核心邏輯修改或複雜重構）委派 `reviewer` 針對目前程式碼現況與計畫內容進行審查，並提供審查報告。
- **【人類審核】**：主動提示使用者「計畫與預審報告已完成，請審核」，等待人類工程師審核。
- 若人類工程師提出建議，必須按照建議修改 `plan.md`。
- 只有在獲得人類工程師放行後，才能繼續執行下一步。

### 4. 任務實作 (Execution Phase)
- 開始實作任務，並將詳細的開發進度、變動與執行紀錄寫在專案任務管理目錄中的 `current-note.md` 中。
- 嚴格遵守專案的程式風格與規範。
- 若實作過程中出現技術難題或發現潛在風險，必須記錄在 `current-note.md` 並告知使用者。

### 5. 程式測試工作 (Testing)
- **參考技能**：啟動 `program-testing` 技能進行驗證。
- **任務範疇**：包含撰寫單元測試、整合測試以及執行自動化測試套件，確保功能正確與系統穩定。
- **環境相容性**：特別針對 Java Spring Boot 專案，當增加單元測試框架（如引入 `spring-boot-starter-test`）時，必須同步檢查專案中的 `Dockerfile`。確保 Docker 啟動指令（如 `ENTRYPOINT` 或 `CMD`）或構建流程中，不會觸發單元測試執行（例如應使用 `-x test` 跳過測試），以確保容器能快速且直接地啟動應用程式服務。

### 6. 程式審查工作 (Post-Review)
- **委派對象**：一律委派 `reviewer` 執行。
- **任務範疇**：在實作完成後執行最終的 Code Review，確保程式碼品質、架構優雅性與系統安全性。
- **報告產出**：完成審查後，必須向人類工程師彙報審查報告內容，並等待確認。

### 7. 審查後修正 (Revision Phase)
- 若在 Reviewer 審查後需要修改程式，必須先閱讀專案任務管理目錄中的 `review-note.md`（或 Reviewer 的意見）。
- 理解審查建議與需要修正的點後，再進行程式碼修改。
- 修改完成後，再次依循測試與審查流程。
